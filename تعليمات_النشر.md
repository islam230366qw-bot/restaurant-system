# تعليمات نشر نظام إدارة المطعم على Cloudflare

## المتطلبات
- Node.js 18+
- حساب Cloudflare
- Wrangler CLI (مثبت مع npm)

---

## 1. تثبيت الاعتماديات

افتح Terminal في مجلد `C:\Users\islam\OneDrive\Desktop\سيستم` ونفذ:

```bash
cd worker
npm install
cd ..
cd frontend
npm install
cd ..
```

---

## 2. تسجيل الدخول إلى Cloudflare

```bash
npx wrangler login
```

---

## 3. إنشاء قاعدة البيانات D1

```bash
npx wrangler d1 create restaurant-db
```

**مهم:** بعد تنفيذ الأمر، سيظهر لك `database_id`. انسخه وضعه في ملف `wrangler.toml` في السطر:
```
database_id = "القيمة_التي_ظهرت"
```

---

## 4. تنفيذ هيكل قاعدة البيانات

```bash
npx wrangler d1 execute restaurant-db --file=./schema.sql --remote
```

---

## 5. إنشاء R2 bucket

```bash
npx wrangler r2 bucket create restaurant-images
```

---

## 6. إنشاء أول مستخدم (مدير)

أولاً، استخدم موقع https://bcrypt-generator.com لتوليد كود Hash لكلمة المرور.

مثال: إذا كانت كلمة المرور `admin123`، ضعها في الموقع وانسخ الـ Hash الناتج.

ثم نفذ:

```bash
npx wrangler d1 execute restaurant-db --remote --command="INSERT INTO users (username, password_hash, full_name, role) VALUES ('admin', 'HASH_الخاص_بك', 'مدير النظام', 'manager');"
```

---

## 7. نشر الـ API (Worker)

```bash
npx wrangler deploy
```

بعد النشر سيظهر رابط مثل: `https://restaurant-api.your-subdomain.workers.dev`

---

## 8. بناء ونشر الواجهة الأمامية

```bash
cd frontend
npm run build
npx wrangler pages deploy ./dist
```

اختر اسم المشروع (مثلاً `restaurant-system`)

---

## 9. ربط النطاق (اختياري)

من لوحة Cloudflare Dashboard:
- اذهب إلى Workers & Pages
- اختر Pages project
- اذهب إلى Custom domains
- أضف نطاقك المخصص

---

## 10. استخدام النظام

1. ادخل على رابط الواجهة (Pages)
2. سجل دخول بـ: `admin` وكلمة المرور اللي اخترتها
3. ابدأ بإضافة:
   - إعدادات المطعم
   - التصنيفات
   - أصناف المنيو
   - الموظفين
4. من الإعدادات، أضف موظفين جدد بدور "كاشير" عشان يستخدموا شاشة الطلبات

## ملاحظات مهمة

1. **JWT_SECRET**: في `wrangler.toml` غير القيمة الافتراضية لسر قوي
2. **Database ID**: لازم تحط `database_id` بعد ما تنشئ D1
3. **أول مستخدم**: ينشئ يدويًا عبر SQL (مش من الواجهة)
4. **الصور**: الصور المرفوعة تظهر عبر route `/api/upload/:key`
5. **لا توجد بيانات وهمية**: كل الجداول فارغة، تضيف البيانات بنفسك من لوحة التحكم
6. **النسخ الاحتياطي**: استخدم `npx wrangler d1 backup create restaurant-db`
